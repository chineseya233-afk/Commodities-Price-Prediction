# 大宗商品采购价格预测 SaaS 系统用户手册

本系统是一个大宗商品采购价格预测 POC 看板，当前商品为 `0# 柴油`。系统由 FastAPI 后端、Vue 3 前端、ECharts 图表、多个预测模型和 AI 分析服务组成。后端启动后会自动加载价格数据、训练/运行模型、生成综合预测，并把前端页面托管在同一个服务中。服务保持运行时，后台会持续检查 EIA/FRED 数据和新闻情绪，数据更新后会自动刷新预测与报告。

## 1. 快速启动

### 1.1 环境要求

- Python 3.10 或更高版本
- Chrome / Edge / Firefox 浏览器
- Windows、macOS、Linux 均可运行
- 可联网时可调用 AI API 和外部数据接口；离线时会使用模拟数据和模板报告

### 1.2 安装依赖

在项目根目录执行：

```powershell
cd "D:\AI Folder\Commodities Price Prediction"
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r backend\requirements.txt
```

如果 `.venv` 已经存在，只需要执行安装依赖命令即可。

### 1.3 启动服务

```powershell
cd "D:\AI Folder\Commodities Price Prediction"
.\.venv\Scripts\python.exe -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

浏览器打开：

```text
http://127.0.0.1:8000
```

当前前端是 Vue 3 CDN 版本，不需要单独启动 npm 或 Vite 服务。FastAPI 会直接托管 `frontend/index.html` 和 `frontend/src` 下的脚本、样式文件。

建议只启动一次并保持服务运行。后端会按 `.env` 中的 `DATA_REFRESH_SECONDS` 周期检查数据源，前端也会定时刷新看板数据。静态前端资源和 API 都带有 no-store 缓存头，浏览器通常不需要手动清缓存。

### 1.4 停止服务

在运行 uvicorn 的终端里按 `Ctrl + C`。

## 3. 如何更换 AI API

所有密钥都放在项目根目录的 `.env` 文件中，不要写进前端代码，不要提交到 Git 仓库。

推荐配置项：

```env
OPENAI_COMPATIBLE_API_KEY=你的模型API密钥
OPENAI_COMPATIBLE_BASE_URL=https://api.deepseek.com/v1
OPENAI_COMPATIBLE_MODEL=deepseek-v4-pro
OPENAI_COMPATIBLE_PROVIDER=deepseek
```

这是一组通用 OpenAI-compatible 接口配置，不绑定具体厂商。DeepSeek 只是当前默认示例。如果要接入其他模型供应商，只需要替换：

- `OPENAI_COMPATIBLE_API_KEY`：模型服务的 API key。
- `OPENAI_COMPATIBLE_BASE_URL`：兼容 OpenAI Chat Completions 协议的 base URL。
- `OPENAI_COMPATIBLE_MODEL`：模型名。
- `OPENAI_COMPATIBLE_PROVIDER`：前端展示用供应商标签，可选。

系统也兼容旧变量 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL`，以及历史 `DEEPSEEK_*` 变量，但新配置建议统一使用 `OPENAI_COMPATIBLE_*`。修改后需要重启后端服务。当前 `.env` 已删除旧厂商专用 key，只保留 DeepSeek key 并迁移到通用接口字段。

页面底部状态栏会显示当前后端实际加载的供应商和模型，例如：

```text
LLM: deepseek: deepseek-v4-pro
```

AI 在系统中的作用：

- 生成“专家研判报告”和“三维度风险报告”。
- 根据 EIA 新闻、价格走势和模型指标生成“新闻调优”。
- 当模型 QA 异常时，当前配置的通用 LLM 会生成修复建议；系统会先用确定性 guardrail 修复或剔除异常模型，AI 建议作为可审计的辅助依据。
- AI 输出不会直接覆盖 `p10/p50/p90` 原始预测值，数值调优会经过后端规则约束。

安全说明：

- `.env` 已被 `.gitignore` 忽略，默认不会进入 Git。
- AI token、数据库地址、登录密钥只在后端读取。
- 前端浏览器只拿到分析结果和预测数据，不应该拿到任何 API token。
- 如果要部署到公网，请务必把 `.env.example` 中的 `JWT_SECRET_KEY` 换成强随机字符串，并把 `COOKIE_SECURE=true`。

## 4. 如何更换数据源

当前启动链路在 `backend/main.py` 中：

1. 先尝试 EIA 数据源：`backend/data_providers/eia_provider.py`
2. 再尝试 FRED 宏观数据：`backend/data_providers/fred_provider.py`
3. 如果真实接口不可用，回退到中国柴油模拟器：`backend/data_providers/simulator.py`
4. 数据经过 `DataPreprocessor` 清洗，再进入特征工程和模型预测

相关 `.env` 配置：

```env
EIA_API_KEY=demo
EIA_BASE_URL=https://api.eia.gov/v2
EIA_DIESEL_SERIES=EER_EPD2DXL0_PF4_Y35NY_DPG
FRED_API_KEY=
FRED_BASE_URL=https://api.stlouisfed.org/fred
MARKET_DATA_START_DATE=2006-06-01
DATA_REFRESH_SECONDS=300
DATABASE_URL=sqlite+aiosqlite:///./data/commodity_prediction.db
```

当前真实数据处理规则：

- EIA 提供柴油美元/加仑价格。
- FRED 提供宏观变量，包括 `DEXCHUS` 人民币兑美元汇率、Brent 油价和联邦基金利率等。
- 后端会用 FRED 汇率把 EIA 柴油价格换算为 RMB/吨，再用于训练、回测和前端展示。
- 官方数据源可能滞后几天。例如系统日期是 `2026-06-06`，但 EIA 最新价格可能只到 `2026-06-01`，这属于数据源发布时间差异，不代表系统日期错误。
- `/api/health` 中的 `data_source=eia` 表示当前正在使用真实 EIA 数据；如果接口失败，会显示 simulator 并回退模拟数据。

如果要换成自己的价格数据，有两种方式：

1. 新增一个 provider，例如 `backend/data_providers/local_csv_provider.py`，返回包含 `date`、`price`、`high`、`low` 等字段的 DataFrame。
2. 在 `backend/main.py` 的初始数据加载逻辑里，把 `price_df` 改为读取你的 provider。

建议数据格式：

| 字段 | 说明 |
| --- | --- |
| `date` | 日期，建议 `YYYY-MM-DD` |
| `price` | 柴油价格，单位 RMB/吨 |
| `high` | 当日高价，没有可用 `price` 代替 |
| `low` | 当日低价，没有可用 `price` 代替 |
| 其他字段 | 可加入库存、汇率、原油价格、政策事件标签等特征 |

## 5. 训练、验证和测试集如何工作

系统不再固定只使用 2024-2025 数据训练，也不再固定用 2026 年 1-4 月验证。当前逻辑会尽量使用 EIA/FRED 能拿到的全部历史数据，并按时间顺序切成三段：

1. 训练集：最早可用历史数据到验证集之前。
2. 验证集：训练集之后的一段最新历史窗口，用于选择最优模型。
3. 测试集：最后一段 holdout 数据，只用于报告最终泛化表现，不参与模型选择。

当前实现位置：

```text
backend/main.py::_evaluate_fixed_train_test_split()
```

切分规则：

- 测试集长度通常为最近 30-90 天，数据越多窗口越长，上限 90 天。
- 验证集长度通常也是 30-90 天。
- 数据足够时，训练集和验证/测试集之间会保留 1 条记录的 embargo 间隔，降低时间泄漏风险。
- `best_model` 由验证集 MAPE 选出。
- `oracle_test_best_model` 只是诊断项，表示如果偷看测试集会选哪个模型，不能作为真实选型依据。
- `overfit_guard` 会报告验证集和测试集 MAPE 差距，用来提示是否存在泛化风险。

页面“固定验证”展示的含义：

```text
训练窗口 -> 验证窗口 -> 测试窗口
验证集最优价格模型
反过拟合校验：通过 / 需关注泛化差距
```

如果要控制训练历史起点，可以修改 `.env`：

```env
MARKET_DATA_START_DATE=2006-06-01
```

原则：不要把测试集结果用于调参或选模型，否则模型指标会虚高。

## 6. 预测日期和今日线如何理解

预测数组必须和“最新真实价格日期”绑定，而不是和“今天”绑定。

正确规则是：

- `p50[0]` 表示“最新真实价格日期的下一天”。
- 如果 EIA 最新价格只更新到 `2026-06-01`，那么 `p50[0]` 对应 `2026-06-02`。
- 如果系统日期是 `2026-06-06`，前端会同时显示：
  - 预测起点线：`2026-06-02`
  - 今日线：`2026-06-06`
- 如果以后数据源刚好更新到昨天，使预测起点和今日落在同一天，前端会画两条带像素偏移的日期标记线，不会再合并成“今日 / 预测起点”一条线。
- 接口返回的预测点里包含 `source_index`，用于排查某个显示日期对应模型输出数组的哪个下标。

看板上的 30 天、60 天、90 天按钮只控制历史价格窗口长度；预测仍按 30 天输出，并按“精确预测 1-7 天、标准预测 8-14 天、趋势参考 15-30 天”分层展示。

## 7. 各模型怎么看和怎么用

### Naive

基线模型。主要用于判断其他模型是否真的比“沿用最近价格”更好，不建议作为最终采购决策模型。

Naive 通常预测一条平线，因此“方向准确率”可能显示为 `N/A` 或在后端标记为不适用。这不是系统故障，而是因为平线基线没有真实的看多/看空方向。当前系统会把 Naive 从综合模型选择中剔除，避免它干扰最终决策。

### Prophet

趋势和季节性模型。适合观察中期趋势、节奏变化和价格均值回归，但遇到突发政策或新闻冲击时可能反应较慢。

如果 Prophet 的“区间覆盖率”为 0，表示在最近 holdout 验证里，实际价格没有落入它给出的 p10/p90 区间。这通常说明区间过窄或市场波动超出该模型假设。系统不会强行美化这个指标，会在综合模型中优先选择覆盖率更合适的模型。

### XGBoost

机器学习特征模型。使用价格滞后项、滚动统计、宏观变量等特征。适合捕捉非线性关系，短期方向判断通常更有参考价值。

### TFT

时间序列深度学习模型。若 PyTorch Forecasting 环境可用，会尝试使用训练好的 TFT；如果依赖或模型文件不可用，会自动进入统计 fallback，不会中断系统。

### Ensemble 综合模型

综合模型不是简单平均，而是按预测区间分段选模型：

- 1-7 天：选择该短期段内价格、方向、区间覆盖表现最好的模型
- 8-14 天：重新选择该中期段内最优模型
- 15-30 天：重新选择远期趋势参考模型

综合预测会分别取：

- 价格预测：使用价格误差最低的模型
- 方向判断：使用方向准确率最高的模型
- 区间范围：使用区间覆盖率接近目标且区间不过宽的模型
- AI 调优：结合近期新闻、价格走势和风险因素做数值微调

区间覆盖不是越高越好。100% 覆盖率可能只是区间太宽，采购参考价值反而下降。系统会优先选择接近目标覆盖率且区间宽度合理的模型。

## 8. QA 校验和自动修复

每个模型输出后会经过硬规则 QA：

- 预测值必须为正数。
- 价格必须处于合理范围。
- 单日涨跌幅不能异常。
- 7 日累计偏差不能异常。
- 预测值不能偏离历史波动范围。
- `p10 < p50 < p90` 的区间顺序必须正确。
- 区间宽度不能过窄或过宽。

如果模型失败：

1. 后端会先尝试确定性 guardrail 修复，例如限制短端累计偏差、修正区间宽度。
2. 修复后重新跑 QA。
3. 仍不通过的模型会被剔除出 ensemble。
4. 当前配置的通用 LLM 可生成模型修复建议，但不会绕过规则校验直接让异常模型参与决策。

页面“数据质量检查结果”中：

- `通过` 表示所有参与模型满足该项规则。
- `异常 xxx(已修复)` 表示模型曾经失败但已自动修复。
- `异常 xxx(已剔除)` 表示模型未参与最终综合预测。

## 11. 常见问题

### 为什么今天是 2026-06-06，但最新价格是 2026-06-01？

这是 EIA 数据源发布时间滞后造成的。系统日期和最新真实价格日期是两个概念。预测从最新真实价格的下一天开始，今日线单独标记当前日期。

### 为什么 Prophet 区间覆盖率是 0？

表示最近验证窗口内实际价格没有落入 Prophet 的 p10/p90 区间。系统会保留这个真实指标，并选择覆盖率更合适的模型提供综合预测区间。

### 为什么 Naive 方向准确率是 N/A 或 0？

Naive 是平线基线，没有方向预测能力。后端会标记为不适用并从综合模型中剔除。它仍保留在模型对比中，用于衡量其他模型是否超过简单基线。

### AI 策略建议内容很长怎么办？

决策大屏第二页的 AI 策略建议支持滚动，不会再用省略号截断。风险研判区域也支持滚动。

### 数据会不会持续更新？

会。只要后端服务不关闭，后台会按 `DATA_REFRESH_SECONDS` 周期检查 EIA/FRED 和新闻情绪。数据发生变化后会自动重建预测和报告。
