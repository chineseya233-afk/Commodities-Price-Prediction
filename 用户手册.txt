大宗商品采购价格预测 SaaS 系统用户手册

1. 快速启动

项目根目录：
D:\AI Folder\Commodities Price Prediction

安装依赖：
cd "D:\AI Folder\Commodities Price Prediction"
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r backend\requirements.txt

启动服务：
cd "D:\AI Folder\Commodities Price Prediction"
.\.venv\Scripts\python.exe -m uvicorn backend.main:app --host 0.0.0.0 --port 8000

浏览器打开：
http://127.0.0.1:8000

当前前端由 FastAPI 托管，不需要单独启动 npm 或 Vite。建议启动一次后保持服务运行，后端会按 DATA_REFRESH_SECONDS 持续检查 EIA/FRED 数据和新闻情绪。

2. 登录账号

admin / Admin123456@：管理员
executive / Exec123456@：决策用户
procurement / Proc123456@：采购用户

刷新页面不会强制退出。只有会话过期、主动退出、关闭页面超过有效期后重新进入，才需要重新登录。

3. AI API

在 .env 中配置：
OPENAI_COMPATIBLE_API_KEY=你的模型API密钥
OPENAI_COMPATIBLE_BASE_URL=https://api.deepseek.com/v1
OPENAI_COMPATIBLE_MODEL=deepseek-v4-pro
OPENAI_COMPATIBLE_PROVIDER=deepseek

这是通用 OpenAI-compatible 接口配置，不绑定具体厂商。DeepSeek 是当前默认示例；如果要换其他模型，只替换 API key、base URL 和 model 即可。系统也兼容旧的 LLM_API_KEY/LLM_BASE_URL/LLM_MODEL 和 DEEPSEEK_* 变量，但建议新配置统一使用 OPENAI_COMPATIBLE_*。

当前 .env 已删除旧厂商专用 key，只保留 DeepSeek key 并迁移到通用接口字段。模型接口不可用时，系统使用本地模板报告，不影响基础预测图表。

AI 负责生成专家研判、三维度风险报告、新闻调优和模型异常修复建议。AI 修复建议由当前配置的通用 LLM 生成，不会直接绕过后端规则修改预测值。

4. 数据源

当前数据链路：
先尝试 EIAProvider，再合并 FREDProvider 宏观数据；如果真实接口不可用，就使用 ChinaDieselSimulator 模拟数据。

相关文件：
backend/data_providers/eia_provider.py
backend/data_providers/fred_provider.py
backend/data_providers/simulator.py

相关 .env：
EIA_API_KEY=你的EIA密钥
EIA_BASE_URL=https://api.eia.gov/v2
EIA_DIESEL_SERIES=EER_EPD2DXL0_PF4_Y35NY_DPG
FRED_API_KEY=你的FRED密钥
FRED_BASE_URL=https://api.stlouisfed.org/fred
MARKET_DATA_START_DATE=2006-06-01
DATA_REFRESH_SECONDS=300

EIA 提供柴油美元/加仑价格，FRED 提供 DEXCHUS 汇率等宏观变量。后端会换算为 RMB/吨。官方数据可能滞后几天，例如系统日期是 2026-06-06，但 EIA 最新价格只到 2026-06-01，这是数据源发布时间差异，不是系统日期错误。

5. 训练、验证和测试

系统会尽量使用全部历史数据，不再局限于 2024 年或固定 2024-2025 窗口。

当前切分方式：
训练集：最早可用历史数据到验证集之前。
验证集：训练集之后的一段最新历史窗口，用于选模型。
测试集：最后一段 holdout，只用于报告最终泛化表现。

best_model 来自验证集 MAPE。oracle_test_best_model 只是诊断项，不能用于真实选型。overfit_guard 会提示验证集和测试集 MAPE 差距。

6. 预测日期规则

预测数组和“最新真实价格日期”绑定，不和“今天”绑定。

p50[0] = 最新真实价格日期的下一天。

例如最新真实价格是 2026-06-01：
p50[0] 对应 2026-06-02。
如果今天是 2026-06-06，看板会同时显示预测起点线 2026-06-02 和今日线 2026-06-06。

如果以后预测起点和今日落在同一天，前端会画两条带像素偏移的线，不会合并成“今日 / 预测起点”一条线。

7. 模型说明

Naive：
平线基线模型，用于衡量其他模型是否超过简单沿用最近价格。Naive 没有真实方向预测能力，方向准确率可能显示 N/A 或在后端标记为不适用，并会从综合模型中剔除。

Prophet：
趋势和季节性模型。若区间覆盖率为 0，表示最近验证窗口中实际价格没有落入 Prophet 的 p10/p90 区间。系统会保留真实指标，不会伪造覆盖率。

XGBoost：
特征机器学习模型，使用价格滞后项、滚动统计、宏观变量等特征。

TFT：
时间序列深度学习模型。若环境或模型文件不可用，会自动使用统计 fallback，系统不会中断。

Ensemble：
不是简单平均。系统会按 1-7 天、8-14 天、15-30 天分别选择价格误差、方向准确率和区间覆盖更合适的模型。区间覆盖率不是越高越好，系统优先选接近目标覆盖率且区间不过宽的模型。

8. 页面说明

决策大屏第一页：
历史价格、预测区间、新闻调优、预测起点线、今日线。

决策大屏第二页：
三维度风险报告和 AI 策略建议。内容较长时支持滚动，不会用省略号截断。

采购看板：
模型指标、固定验证结果、预测明细、采购动作和 AI 问答。

30 天、60 天、90 天按钮只控制历史价格窗口长度。预测仍按 30 天输出，并分为精确预测 1-7 天、标准预测 8-14 天、趋势参考 15-30 天。

9. QA 和自动修复

模型输出会经过规则 QA：
预测值正数、价格合理范围、单日涨跌幅、7 日累计偏差、历史波动范围、区间顺序、区间宽度。

如果模型失败：
后端先尝试确定性 guardrail 修复；修复后重新 QA；仍失败则剔除出 ensemble；当前配置的通用 LLM 可生成修复建议，但不会绕过规则校验。

10. 常用接口

/api/health：健康检查
/api/auth/login：登录
/api/auth/session：恢复登录态
/api/dashboard/summary：看板总数据
/api/predictions/latest?model=ensemble：指定模型预测
/api/predictions/all-models：全部模型预测
/api/metrics/comparison：模型性能对比
/api/backtest/fixed-split：动态训练/验证/测试三段评估
/api/backtest/results：滚动回测
/api/chat：AI 采购助手
