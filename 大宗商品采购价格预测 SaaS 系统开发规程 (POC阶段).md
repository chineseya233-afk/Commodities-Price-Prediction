# 大宗商品采购价格预测 SaaS 系统开发规程 (POC阶段)

## 1. 项目概述

本系统旨在为福建供销集团提供大宗原材料的价格预测与决策辅助服务。系统采用"大模型 + 小算法"的融合架构，实现从数据抓取、质量控制、算法预测到智能解读的全链路闭环，最终输出适用于高管决策与采购执行的双视角可视化看板。

### 1.1 POC 范围界定
> **POC 聚焦品种：柴油（0#柴油）**
> - **目标列：** 柴油每日报价平均值
> - **观测窗口：** 历史 30 天滑动窗口
> - **预测horizon：** 未来 30 天逐日预测（分三层精度）
> - **核心验收指标：** 精确周期 MAPE ≤ 3%，方向准确率 ≥ 70%
> - **交付物：** 可运行的 Web 端 SaaS 系统（含预测引擎、QA 校验、LLM 解读、可视化看板）

### 1.2 预测周期分层架构（V2 新增）
> ⚠️ **核心设计变更：预测周期从 7 天扩展到 30 天，并按精度分三层管理。**
>
> | 区间 | 名称 | 天数 | 置信区间特征 | 可信度 | 用途 |
> |------|------|------|-------------|--------|------|
> | T+1 ~ T+7 | **精确预测** | 7天 | 窄区间（P10-P90 差值 ≤ 3%） | ★★★★★ | 精确采购决策 |
> | T+8 ~ T+14 | **标准预测** | 7天 | 中等区间（差值 3%-6%） | ★★★☆☆ | 库存计划参考 |
> | T+15 ~ T+30 | **模糊预测** | 16天 | 宽区间（差值 6%-15%） | ★★☆☆☆ | 趋势方向参考 |
>
> **设计原则：**
> - 置信区间随预测天数自然扩散，前端图表用不同颜色深浅区分三个区间
> - 精确区间使用实线，标准区间使用虚线，模糊区间使用点线
> - 每层区间标注可信度等级，帮助决策者判断参考权重

### 1.3 技术栈选型
| 层级 | 技术选型 | 选型理由 |
|------|---------|---------| 
| **后端 API** | FastAPI (Python 3.10+) | 原生异步支持、自动 OpenAPI 文档、高性能 |
| **预测引擎** | PyTorch Forecasting (TFT) + XGBoost + Prophet | 多变量时序预测 SOTA + 基线对比 |
| **基线模型** | Prophet / XGBoost / Naive | 作为 TFT 的对比基线，验证复杂模型的增量价值 |
| **数据库** | SQLite (POC) → PostgreSQL + TimescaleDB (生产) | POC 轻量化，生产环境无缝升级 |
| **LLM 集成** | OpenAI-compatible 通用接口（默认示例 DeepSeek） | 智能分析报告 + 风险研判 + QA 软校验 |
| **前端** | 原生 HTML/CSS/JS + Apache ECharts | 轻量化 POC，后续可迁移至 Next.js |
| **部署** | Docker Compose | POC 阶段一键部署 |

## 2. 核心架构与阶段拆解

### 第一阶段：多源数据底座建设 (Data Engineering)
**目标：** 构建物理隔离、安全合规的数据中台，支撑 POC 验证。

#### 1.1 外部真实数据接入
通过 API 或定制爬虫接入"唐宋网"、"我的钢铁网"等行业权威数据平台。
* **核心采集指标：** 燃油每日基准价、钢材社会库存总量、全国及重点区域高炉开工率、主流钢材利润水平、海外港口发货量等。

#### 1.2 POC 数据降级策略
> ⚠️ 由于行业数据平台（卓创资讯、隆众资讯、唐宋网）均为企业级付费服务，POC 阶段采用以下降级策略：
> 1. **第一优先级 — 公开权威数据：** 接入美国 EIA (Energy Information Administration) 柴油及原油现货价格数据（免费 API，日频），以及 FRED 经济数据库宏观指标
> 2. **第二优先级 — 国内公开数据：** 抓取国家发改委成品油调价公告、上海期货交易所燃油期货数据
> 3. **第三优先级 — 高保真仿真数据：** 基于真实市场统计分布特征，使用蒙特卡洛模拟生成符合实际波动特征的中国柴油价格数据（含均值回归、季节性、跳跃扩散等特性）
> 4. **接口标准化：** 所有数据源均通过统一的 `DataProvider` 抽象接口封装，正式合作后可无缝切换至付费数据源

#### 1.3 内部仿真数据构建 (Mock)
在系统中建立标准化的内部数据接口结构，模拟生成符合业务逻辑的采购订单明细、历史库存量、合同履约情况等字段。此举旨在确保系统架构完整，待正式合作后可无缝替换为真实 ERP 数据。

#### 1.4 数据质量基线
* **缺失值监控：** 建立数据完整度仪表盘，关键特征缺失率阈值 ≤ 5%
* **异常值检测：** 基于 IQR / Z-Score 的自动异常标记管道
* **数据血缘追踪：** 记录每条数据的来源、采集时间、清洗规则，便于回溯

### 第二阶段：核心预测引擎开发 (Algorithmic Modeling)
**目标：** 部署并训练高精度的多变量时序预测模型，支撑 30 天分层预测。

#### 2.1 核心算法：Temporal Fusion Transformer (TFT)
* **选型依据：** 专为多变量时序预测设计，原生支持处理数值型（库存、利润）、时序型（价格序列）及类别型特征，且天然适应大宗商品数据中常见的"缺失值"与"不规则采样"问题。
* **关键优势：**
    * 内置注意力机制可解释性 —— 输出变量重要性排序，辅助交易员理解预测逻辑
    * 原生支持概率预测（分位数输出），直接生成预测区间而非单一点预测
* **POC 部署：** 使用 `pytorch-forecasting` 库 + RTX 2070 Laptop GPU 进行训练，当 GPU 不可用时自动降级到统计方法回退

#### 2.2 基线模型对比矩阵
> 为验证 TFT 复杂度的必要性，POC 阶段将同时训练以下基线模型进行对比：
> | 模型 | 复杂度 | 用途 |
> |------|--------|------|
> | **Naive Forecast** (随机游走) | 极低 | 绝对下限基准 |
> | **Prophet** | 低 | 验证季节性+趋势分解的效果 |
> | **XGBoost** (特征工程) | 中 | 验证传统ML在结构化特征上的表现 |
> | **TFT** | 高 | 核心模型，需证明对基线有显著提升 |

#### 2.3 开发与训练流程
1. **数据预处理：** 将时序数据聚合，处理缺失值（如前向填充+线性插值组合策略）。
2. **特征工程配置：** 设定目标列为"柴油每日报价平均值"，观测周期设为历史 30 天，预测周期设为未来 30 天。
    * **已知未来变量：** 日历特征（星期几、是否节假日、月份）、计划性政策事件、发改委调价窗口
    * **未知未来变量：** 原油价格、库存水平、炼厂开工率
    * **静态变量：** 品种类别、区域标识
3. **超参优化：** 引入 Optuna 机制进行超参寻优（包含隐藏层节点数、注意力头数、学习率、dropout率等），确保模型 MAPE 控制在行业最优水平。

#### 2.4 30天分层预测实现（V2 新增）
> **所有模型统一输出 30 天预测，系统自动按精度分层：**
> 1. **精确预测 (T+1~T+7)：** 置信区间窄，直接用于采购决策
> 2. **标准预测 (T+8~T+14)：** 置信区间递增，用于库存规划
> 3. **模糊预测 (T+15~T+30)：** 置信区间宽，仅供趋势方向参考
>
> **置信区间扩散机制：**
> - 精确区间：P10-P90 差值 ≤ 基准价的 3%
> - 标准区间：差值自然扩散至 3%-6%
> - 模糊区间：差值扩散至 6%-15%（允许宽幅波动）

#### 2.5 回测验证框架
> 采用滑动窗口回测（Walk-Forward Validation），而非简单的 train/test 划分：
> * **回测窗口：** 最近 90 天，每次向前滑动 7 天
> * **评估指标集：**
>     * MAPE（平均绝对百分比误差）— 精度衡量
>     * RMSE（均方根误差）— 大误差惩罚
>     * 方向准确率（Directional Accuracy）— 涨跌判断正确率
>     * 覆盖率（Coverage Rate）— 95% 预测区间包含真实值的比率
> * **可视化输出：** 每轮回测生成预测 vs 真实值对比曲线

#### 2.6 概率预测与置信区间
> 作为资深交易员的核心建议：**单一点预测对决策者毫无意义，必须输出预测区间。**
> * 输出 10%、50%（中位数）、90% 分位数预测
> * 基于分位数生成"乐观/基准/悲观"三情景分析
> * 在前端以扇形区域图展示置信带

### 第三阶段：AI 流程质量检测 (Quality Assurance)
**目标：** 在算法计算层与业务展现层之间构建智能"数字防火墙"，确保输出逻辑的严密性与准确性。

#### 3.1 QA 智能体 (QA LLM Agent)
部署独立的校验大模型节点。

#### 3.2 执行策略与检测维度
* **数值异常拦截：** 实时比对预测值与历史同期基准，若波动幅度触发系统熔断阈值（如日涨跌幅超过 ±5%），则自动拦截异常结果并记录日志。
* **行业逻辑一致性校验：** 结合行业常识进行多因子逻辑互斥检测。例如：若模型预测柴油价格暴涨，但同时输入的原油价格数据呈现断崖式下跌，QA 智能体需判定预测结果存在逻辑冲突，并触发重新拟合。
* **数据完整性熔断：** 监控外部数据源健康度，若关键特征连续多日缺失，触发数据治理告警。

#### 3.3 规则引擎 + LLM 双层校验
> QA 不应完全依赖 LLM 的主观判断，采用双层架构：
> * **第一层（硬规则引擎）：** 基于参数化阈值的确定性校验，零误判
>     * 预测值超出历史极值 ±2σ → 自动拦截
>     * 7日累计涨跌幅超过 ±15% → 告警
>     * 预测值出现负数或零值 → 直接拒绝
>     * 置信区间排序校验：P10 ≤ P50 ≤ P90
>     * 区间宽度合理性检查：0.5%-20% 范围
> * **第二层（LLM 软校验）：** 处理需要行业推理的复杂逻辑互斥
>     * 多因子关联合理性评估
>     * 与近期重大政策/事件的一致性判断

#### 3.4 QA 触发风险报告（V2 新增）
> **当 QA 检测发现预测数据可能存疑时，自动生成可读性强的风险条目：**
> - QA 校验失败的模型 → 生成"模型预测风险"条目（非技术语言描述）
> - 多模型预测方向不一致 → 生成"市场分歧风险"条目
> - 置信区间异常偏窄/偏宽 → 生成"不确定性评估"条目

### 第四阶段：大模型智能分析与业务解读 (Generative Insights)
**目标：** 依托大语言模型，将量化预测数据转化为具备专家视角的自然语言报告。

#### 4.1 智能体集成
通过 `OPENAI_COMPATIBLE_API_KEY`、`OPENAI_COMPATIBLE_BASE_URL`、`OPENAI_COMPATIBLE_MODEL` 接入 OpenAI-compatible 大语言模型。POC 默认示例为 DeepSeek，但实现不绑定具体厂商；只要供应商兼容 OpenAI Chat Completions 协议，就可以直接替换 base URL、模型名和 API key。

#### 4.2 Prompt 模板工程
* **角色设定：** 资深能源行业量化分析专家与数据解读专家（POC聚焦柴油，角色描述需匹配）。
* **自动化指标计算：** 指令大模型基于验证数据集，严格计算绝对误差与相对误差百分比。
* **趋势一致性分析：** 从涨跌幅方向、波动周期（峰值/谷值匹配度）多维度研判预测与真实的吻合度。

#### 4.3 结构化输出规范
> LLM 输出必须遵循固定的 JSON Schema，确保前端可解析：
> ```json
> {
>   "summary": "一句话核心结论",
>   "trend_analysis": "趋势分析段落（30天多周期视角）",
>   "risk_factors": ["风险因子1", "风险因子2"],
>   "procurement_advice": {
>     "action": "建仓/观望/减仓",
>     "confidence": "高/中/低",
>     "reasoning": "决策依据",
>     "suggested_price_range": "建议采购价格区间",
>     "timing": "建议执行时机"
>   },
>   "data_quality_notes": "数据质量提示"
> }
> ```

#### 4.4 风险研判报告系统（V2 新增）
> ⚠️ **这是面向决策领导的核心功能升级。**
>
> 风险报告不是简单的系统告警，而是**专业的、分点格式的风险研判报告**，内容清晰易懂，帮助非技术背景的决策者快速理解关键风险。
>
> **报告由三个维度聚合生成：**
>
> ##### 维度一：市场舆情风险（新闻分析）
> - 通过当前配置的通用 LLM 分析大宗商品领域的实时新闻热点
> - 评估 OPEC 政策、国际地缘冲突、汇率波动、库存数据等对柴油价格的潜在影响
> - 每条标注风险等级（高/中/低）+ 具体影响描述
>
> ##### 维度二：模型预测风险（QA 校验触发）
> - 当 QA 检测发现预测数据可能存疑时自动生成
> - 用非技术语言解释为什么预测可能不准（如"模型对近期波动的估计可能不足"）
> - 建议决策者增加人工研判权重
>
> ##### 维度三：数据与政策风险
> - 发改委成品油调价窗口监控
> - 数据源健康度告警
> - 季节性需求变化（农忙/冬储/交通高峰等）
>
> **报告输出格式（分点编号、严谨清晰）：**
> ```text
> 📊 风险研判报告 — 2026年4月12日
> ━━━━━━━━━━━━━━━━━━
>
> ▶ 一、市场舆情风险（新闻分析）
>   1. 【高风险】OPEC+ 减产协议存变数，布伦特原油承压
>      → 影响：若 OPEC+ 增产，国内柴油到岸成本下降 2-3%
>   2. 【中风险】美联储加息预期升温，美元走强
>      → 影响：人民币承压，进口成本上行约 1%
>   3. 【低风险】春耕季节柴油需求企稳
>      → 影响：区域性需求支撑，价格下跌空间有限
>
> ▶ 二、模型预测风险（QA 校验）
>   4. 【预警】XGBoost 模型置信区间偏窄(0.3%)
>      → 解读：模型对未来波动估计不足，实际价格可能超出预测区间
>   5. 【通过】TFT/Prophet 模型 7 项 QA 检查全部通过
>
> ▶ 三、数据与政策风险
>   6. 【关注】发改委调价窗口将在 4月15日 开启
>      → 影响：根据10个工作日调价机制估算，本轮可能小幅上调 50-100 元/吨
> ```

### 第五阶段：SaaS 平台前端展现与可视化工作台 (Platform & Frontend)
**目标：** 交付具备多级权限控制和高交互性的 Web 端产品闭环。

#### 5.1 权限中台 (RBAC)
实现企业级登录与角色鉴权，严格隔离"领导决策视图"与"采购业务视图"。
> **POC 简化方案：** 使用 JWT + 硬编码角色（admin/executive/procurement），暂不对接企业 SSO

#### 5.2 核心业务模块

##### 高管决策大屏 (Executive Dashboard)（V2 重新设计）
> ⚠️ **核心设计原则：去技术化、直观易懂。**
> 领导不一定懂 MAPE 和 RMSE，他们主要看**方向、风险和建议**。
>
> **新布局自上而下：**
>
> ```
> ┌──────────────────────────────────────────────────┐
> │ ① 市场快报（一句话核心结论 + 趋势箭头 + 信号灯）  │
> │   "柴油价格未来7天预计小幅上行，建议适量锁价"       │
> │   🟢 安全  /  🟡 关注  /  🔴 警戒                │
> ├──────────────────────────────────────────────────┤
> │ ② 价格趋势总览（大图）                           │
> │   历史 + 精确预测 + 标准预测 + 模糊预测            │
> │   三段区间用颜色深浅自然过渡                       │
> │   时间范围切换：30天 / 60天 / 90天               │
> ├────────────────────┬─────────────────────────────┤
> │ ③ 风险研判（左半）  │ ④ AI 策略建议（右半）        │
> │ 分点逐条报告        │  动作：逢低加仓              │
> │ 编号+等级+影响      │  置信度：高                  │
> │ 新闻/QA/政策三维度  │  建议价格区间                │
> │                    │  执行时机                    │
> ├────────────────────┴─────────────────────────────┤
> │ ⑤ 简化KPI（去技术化表达）                         │
> │  "预测可信度: 99.4%" + "预计30日涨跌: +2.1%"     │
> └──────────────────────────────────────────────────┘
> ```
>
> **KPI 去技术化表达：**
> - ❌ "MAPE: 0.62%" → ✅ "预测可信度: 99.4%"
> - ❌ "方向准确率: 85%" → ✅ "过去10次中8次预测方向正确"
> - ❌ "Coverage Rate: 92%" → ✅ "价格落在预测区间内的概率: 92%"

##### 采购预测分析看板 (Procurement Ops Terminal)
* **量化追踪：** 以高密度数据表格呈现每日偏差分析及趋势匹配明细。
* **策略赋能：** 展示大模型生成的结构化采购建议，辅助业务端执行"锁量锁价"、"逢低建仓"等具体动作。
* **回测报告面板：** 展示历史预测准确率走势，建立模型信任度。
* **模型性能矩阵：** 展示各模型（TFT/XGBoost/Prophet）的量化评估指标对比。
